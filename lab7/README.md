1. Загальний опис проекту
Метою даної роботи є вивчення механізмів створення, синхронізації та керування потоками виконання (threads) у середовищі Linux. Програма реалізує алгоритм паралельного обчислення суми елементів великого масиву, використовуючи системну бібліотеку `pthread`.

2. Архітектура та логіка програми
Проект демонструє принцип "Divide and Conquer" (розділяй та володарюй):
Масив: Використовується масив цілих чисел розміром 1000 елементів, заповнений одиницями для верифікації результату.
Паралелізм: Задача розподіляється між 4 незалежними робочими потоками. Кожен потік обробляє свій діапазон індексів (по 250 елементів).
Синхронізація: Головний потік (Main Thread) використовує системний виклик `pthread_join()`, щоб дочекатися завершення всіх обчислень перед виведенням фінального результату.

3. Технічні особливості реалізації
Для забезпечення стабільності та швидкодії було застосовано наступні підходи:
Передача аргументів: Кожен потік отримує адресу унікальної комірки масиву `thread_ids`. Це запобігає стану гонитви (**Race Condition**) на етапі ініціалізації.
Безпека даних: Кожен потік записує результат у власну комірку масиву `partial_sums`. Це дозволяє уникнути використання м'ютексів та підвищує продуктивність за рахунок відсутності блокувань.
Системні виклики: На рівні ядра Linux програма взаємодіє з викликами `clone()` (для створення легковагих процесів) та `futex()`.

4. Інструкція із запуску
Збірка проекту автоматизована за допомогою Makefile.

Команди для виконання:
1. Збірка проекту:
   make
2. Запуск програми:
   ./lab7
3. Видалення скомпільованих файлів:
   make clean
